# Руководство по работе с git  
----  
## Основные команды и обозначения  
В командной строке вы всегда находитесь в какой-то папке — просто этого не видно.  
«.» - текущая директория  
«..» - предыдущая директория  
«~» - домашняя директория  
  
**pwd** (print working directory) выводит путь к текущей директории (папке).  
  
**cd** (change directory) – сменить папку. Если в названии папки есть пробелы, нужно писать через кавычки. Чтобы вернуться в родительскую директорию, нужно написать две точки. С помощью одной точки можно обратиться к текущей папке.  
  
**ls** (list contents) – отображает содержимое папки. Если добавить ключ -a, то выведутся и скрытые файлы, начинающиеся с «.»  
  
**touch** – создаёт файл. Пример «touch test.txt»  
  
**mkdir** (make directory) – создаёт директорию. С помощью флага -p можно создать целую структуру папок: mkdir -p dir/dir-inside. Вместе с символами вверху: mkdir ~/my-git-projects  
  
**cp** (copy) – копирование файлов. Принимает два аргумента: что копируем, и куда.  
  
**mv** (move) – по синтаксису то же, что и cp  
  
**cat** (concatenate and print) – выводит содержание (только) текстового файла. После cat надо ввести название файла.
   
**rm** (remove) – удаляет файл. Надо ввести название файла после remove.  
  
**rmdir** (rempve directory) – удаляет пустую папку. Если папка не пустая, то она не будет удалена, и об этом напечается. После rmdir Пишется название папки.  
  
**rm -r** удалит всё содерживое папки и саму папку. Если добавить ключ -f то содержимое удалится без вопросов «а вы точно хотите?». Совмещать ключи можно как -rf  
  
**!!Использование!!** rm и rmdir необратимо, т.е. удалённые файлы и папки исчезают навсегда, потому что не попадают в корзину.  
  
**clip** < путь/к/файлу/ - копирует содержимое файла в буфер обмена.  
  
Можно выполнять писать сразу несколько команд, разделяя их &&  
С помощью стрелочки вверх можно перемещаться по последним командам.  
С помщью tab можно автозаполнение делать.  

## Настройка git  
  
**git config --global** – настройка имени (user.name) и почты (user.email). Пример: git config --global user.name “Feanor Noldor”. С флагом --list выдаст ник и почту.  
  
**ssh-keygen** – генерирует ssh ключ (надо дополнительно смотреть).  
  
## Работа с git  
  
**git init** – делает папку, в которой находишься репозиторием гит. Не стоит создавать один репозиторий внутри другого.  
Чтобы разгитить папку, являющуюся репозиторием, нужно удалить папку .git Именно в ней хранится история изменений.  
  
**git status** – показывает текущее состояние репозитория.  
  
**git add smth.smth** – подготавливает к сохранению файл в репозитории. Ключ --all применяет эту команду ко всем файлам в репозитории. Подготовить можно и всю текущую папку, обратившись к ней через «.». Команда git add лишь запоминает текущее состояние, но не сохраняет его.  
  
Сохранение текущей версии называют коммитом.  
**git commit** – коммитит репозиторий. Если добавить ключ -m то после команды можно написать в кавычках комментарий к этому коммиту.  
  
**git log** – посмотреть историю коммитов.  
  
**git remote add** <имя> <URL> - привязка удалённого репозитория к локальному. Команда выполняется из нужного репозитория. origin – стандартный псевдоним, с помощью которого можно обращаться к главному удалённому репозиторию. Используется для аргумента <имя>. С помощью флага -v можно убедиться, что репозитории связаны.  
   
**git push** – отправляет изменения на удалённый репозиторий. В первый раз эту команду надо вызывать с флагом -u и параматерами имени удалённого репозитория (origin) и название текущей ветки (master, если главная).  
  
**readme.md** – файл «обложка», говорящий о проекте. Является обычным текстовым файлом, но разметка редактируется с помощью языка markdown. Почитать можно [тут](https://gist.github.com/fomvasss/8dd8cd7f88c67a4e3727f9d39224a84c)
